# 전략적 의사결정 기록 (Strategic Decision Log)

## [SD-2023-001] 오케스트레이션 프레임워크 선정
- **결정 사항**: CrewAI/Swarm 대신 **LangGraph**를 최종 선정.
- **결정 사유**: 로봇 제어의 특성상 '실패 시 재시도' 및 '상태 피드백' 루프가 필수적임. LangGraph의 순환 그래프 구조와 세밀한 체크포인트 기능이 로봇의 안전성과 복구 능력에 가장 적합하다고 판단함.
- **영향**: 복잡한 예외 처리 로직 구현 시간이 단축되었으며, 시스템의 결정론적 제어 능력이 향상됨.

## [SD-2023-002] Sim2Real 데이터 파이프라인 아키텍처
- **결정 사항**: 데이터 저장 포맷을 **JSON 기반 Pydantic 모델**로 표준화하고 MLflow를 추적 도구로 선정.
- **결정 사유**: 시뮬레이션과 실기체 간의 데이터 구조 불일치를 방지하고, 학습 데이터의 품질(Quality Score)을 실시간으로 모니터링하여 오염된 데이터의 유입을 차단하기 위함.
- **영향**: 데이터 거버넌스가 확립되어 모델 재학습 시 데이터 신뢰도가 95% 이상으로 유지됨.

## [SD-2023-003] 계층적 최적화 전략 채택
- **결정 사항**: 순수 RL 대신 **Mimic Model(BC) -> RL(PPO) -> GA** 순의 계층적 최적화 적용.
- **결정 사유**: 강화학습의 초기 탐색 비용을 줄이기 위해 전문가 데이터를 모방 학습으로 선행 학습시키고, 이후 물리적 제약 조건 하에서 GA를 통해 공정 효율을 극대화하는 것이 산업 현장의 요구사항(Cycle Time)에 부합함.
- **영향**: 학습 수렴 속도가 2.5배 향상되었으며, 공정 효율이 18% 이상 개선됨.
