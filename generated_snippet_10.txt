# UR5e Sim2Real 상세 기술 아키텍처 명세서

## 1. 시스템 개요
본 시스템은 NVIDIA Isaac Sim(Simulation)과 UR5e 실기체(Real)를 동일한 제어 인터페이스로 운영하는 Sim2Real 아키텍처를 채택한다. LangGraph 오케스트레이터가 전략을 수립하면, RTDE 통신 계층이 이를 실행하고 상태를 피드백한다.

## 2. 컴포넌트 구성
### 2.1. AI Orchestration Layer (LangGraph)
- **Task Planner**: 사용자 목표를 원자적 동작(Atomic Actions)으로 분해.
- **State Manager**: 로봇의 관절 각도, TCP 포즈, 그리퍼 상태를 전역 상태로 관리.

### 2.2. Communication Bridge (Pydantic + RTDE)
- **Data Validator**: Pydantic을 사용하여 입출력 데이터의 유효성 검증 및 타입 보장.
- **RTDE Client**: UR5e의 500Hz 실시간 인터페이스를 통해 제어 명령 송신 및 상태 수신.

### 2.3. Execution Layer (Sim & Real)
- **Sim Driver**: Isaac Sim의 `omni.isaac.urdf` 및 ROS2 Bridge를 통한 가상 제어.
- **Real Driver**: `ur_rtde` 라이브러리를 통한 실기체 직접 제어.

## 3. 기술 스택
- **Language**: Python 3.10+
- **Orchestration**: LangGraph, LangChain
- **Data Validation**: Pydantic v2
- **Robot Interface**: ur_rtde (C++/Python)
- **Simulation**: NVIDIA Isaac Sim / Omniverse
- **Middleware**: ROS2 Humble (Optional for Sensor Fusion)
- **Infrastructure**: Docker, NVIDIA-Container-Toolkit
