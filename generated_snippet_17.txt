# MLOps Data Pipeline Implementation: URSim 관절값 JSON 저장 및 데이터 검증

## 1. MLOps 데이터 파이프라인 개요

본 구현은 URSim 시뮬레이터 또는 실제 UR5e 로봇에서 수집한 관절값 데이터를 MLOps 플랫폼에 통합하기 위한 파이프라인을 구축합니다. 기존의 단순 CSV 로깅을 개선하여, Pydantic 기반의 데이터 유효성 검증을 거친 후 JSON 형식으로 저장하고, MLflow를 사용하여 데이터셋을 버전 관리합니다.

**파이프라인 구성 요소:**

1.  **데이터 수집 (RTDE Client):** URSim에서 실시간 관절값 데이터를 수집합니다.
2.  **데이터 검증 (Data Validation Module):** 수집된 데이터의 스키마 유효성 및 물리적 한계값(Joint Limits)을 검사합니다.
3.  **데이터 저장 (JSON Storage):** 검증된 데이터를 JSON 형식으로 저장합니다.
4.  **MLOps 연동 (MLflow):** 데이터 수집 파이프라인 실행 정보를 추적하고, 저장된 JSON 파일을 데이터셋 아티팩트로 등록합니다.

## 2. Pydantic 기반 데이터 모델 및 데이터 검증 로직

### 2.1. Pydantic 데이터 모델 (`src/data_models/robot_state.py`)

데이터의 스키마를 정의하고, Pydantic의 `Field`를 사용하여 관절값의 물리적 한계(Joint Limits)를 명시적으로 정의합니다. 이는 데이터 수집 단계에서 1차적인 유효성 검증을 수행하는 기반이 됩니다.
