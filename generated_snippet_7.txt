# 에이전트 오케스트레이션 프레임워크 선정 보고서

## 1. 후보군 비교 분석

| 비교 항목 | CrewAI | OpenAI Swarm | LangGraph (선정) |
| :--- | :--- | :--- | :--- |
| **주요 특징** | 역할 기반 협업, Task 중심 | 경량형, 에이전트 간 핸드오버 | 상태 머신 기반, 순환 그래프 지원 |
| **제어 흐름** | 순차적/계층적 (Linear) | 유연한 전이 (Dynamic) | 정밀한 상태 제어 및 루프 (Cyclic) |
| **복잡도 대응** | 중간 | 낮음 | 높음 |
| **로봇 제어 적합성** | 보통 (단순 작업 위주) | 낮음 (실험적 단계) | **매우 높음 (피드백 루프 필수)** |

## 2. 최종 선정: LangGraph

### 선정 사유
1. **순환 그래프(Cycles) 지원**: 로봇 제어는 '인지-판단-실행-확인'의 반복적인 루프가 필수적이다. LangGraph는 순환 구조를 기본적으로 지원하여 재시도 및 피드백 로직 구현에 최적이다.
2. **세밀한 상태 관리(Persistence)**: 체크포인트 기능을 통해 시스템 중단 시에도 마지막 상태에서 복구가 가능하며, 긴 시퀀스의 작업을 안정적으로 관리할 수 있다.
3. **제어권 보장**: 개발자가 에이전트 간의 전이 조건(Edge)을 명확히 정의할 수 있어, LLM의 불확실성을 통제하고 안전한 로봇 기동을 보장할 수 있다.

## 3. 도입 전략
- **Phase 1**: LangGraph를 이용한 기본 'Task Planner'와 'Executor' 구조 설계.
- **Phase 2**: ROS2 인터페이스를 LangGraph의 Tool로 래핑하여 실기체 연결.
- **Phase 3**: 다중 로봇 협업을 위한 Multi-Agent Graph 확장.
