# 에이전트 협업 및 비전 유지 가이드라인

## 1. 단일 진실 공급원 (Single Source of Truth)
- 모든 에이전트는 `config/` 내의 공유 설정 파일을 참조하며, 임의의 하드코딩을 금지한다.
- 하드웨어 사양 변경 시 `orchestration_config.yaml`을 최우선으로 업데이트한다.

## 2. 인터페이스 우선 원칙 (Interface First)
- 새로운 기능을 개발하기 전, Pydantic 스키마(`src/core/schemas.py`)를 먼저 정의하여 에이전트 간 데이터 규격을 맞춘다.

## 3. 안전 우선 (Safety First)
- 모든 실행 에이전트는 `SafetyMonitorAgent`의 승인 없이는 로봇에 명령을 전달할 수 없다.
- 시뮬레이션에서 검증되지 않은 코드는 실기체 배포를 엄격히 금지한다.

## 4. 피드백 루프 활성화
- MLOps 에이전트는 수집된 데이터의 품질 지표를 매주 RL 에이전트에게 공유하여 모델 개선의 근거로 활용한다.
