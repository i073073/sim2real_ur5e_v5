        # DVC에 새로운 데이터 파일 추가
        dvc add data/raw/joint_log_20240523_103000.jsonl

        # Git에 메타데이터 파일(.dvc) 커밋
        git add data/raw/joint_log_20240523_103000.jsonl.dvc .gitignore
        git commit -m "feat: add new joint data from pick-and-place task"
        ```
    3.  DVC 원격 저장소(예: AWS S3, Google Cloud Storage)에 실제 데이터 파일을 업로드합니다.
        ```bash
        # DVC 원격 저장소로 데이터 푸시
        dvc push
        ```
- **기대 효과:**
    - **재현성:** 특정 Git 커밋 해시만 있으면 누구나 동일한 버전의 데이터셋을 정확히 재구성할 수 있습니다.
    - **효율성:** Git 저장소는 가볍게 유지되며, 데이터는 별도의 스토리지에서 효율적으로 관리됩니다.
