### 5.1. 데이터 수집 파이프라인 MLOps 통합 결과

**목표:** UR5e 로봇의 관절값 데이터를 수집하고, 데이터 품질을 검증하며, MLOps 플랫폼을 통해 데이터셋을 관리합니다.

**구현 결과:**

1.  **자동화된 데이터 수집 및 저장:** `data_collection_pipeline.py` 스크립트를 통해 RTDE 클라이언트가 자동으로 실행되어 로봇 데이터를 수집합니다. 수집된 데이터는 JSON 형식으로 저장됩니다.
2.  **데이터 유효성 검증:** `DataQualityManager` 모듈을 통해 Pydantic 스키마 검증 및 물리적 한계값 검증이 실시간으로 수행됩니다. 비정상적인 데이터는 저장 버퍼에서 제외되어, 학습 데이터셋의 품질을 보장합니다.
3.  **MLflow 연동:**
    *   **실험 추적:** 각 데이터 수집 실행은 MLflow의 새로운 Run으로 기록됩니다.
    *   **매개변수 로깅:** 로봇 IP, 수집 시간 등 파이프라인 설정값이 기록됩니다.
    *   **데이터 품질 지표:** `DataQualityManager`에서 생성된 유효/무효 샘플 수, 무효 비율 등의 지표가 MLflow에 기록되어 데이터셋의 신뢰성을 한눈에 파악할 수 있습니다.
    *   **아티팩트 관리:** 최종 JSON 데이터 파일이 MLflow 아티팩트로 저장되어, 데이터셋의 버전 관리가 가능해집니다.

### 5.2. 데이터 거버넌스 및 신뢰성 확보 방안

*   **데이터 스키마 표준화:** Pydantic `RobotState` 모델을 사용하여 `Planning Layer`와 `Execution Layer` 간의 데이터 형식을 표준화했습니다. 이는 `Learning Layer`에서 학습 데이터를 처리할 때 발생하는 데이터 불일치 문제를 사전에 방지합니다.
*   **데이터 품질 게이트:** `DataQualityManager`는 수집된 데이터가 학습에 사용되기 전에 반드시 거쳐야 하는 품질 게이트 역할을 합니다. 이 게이트를 통과하지 못한 데이터는 학습에 사용되지 않으므로, AI 모델의 신뢰성을 높일 수 있습니다.
*   **데이터셋 버전 관리:** MLflow를 통해 수집된 데이터셋은 고유한 Run ID와 함께 아티팩트로 저장됩니다. 이는 특정 모델 학습에 사용된 데이터셋을 정확하게 추적할 수 있게 하여, 재현성(Reproducibility)을 확보합니다.

### 5.3. 향후 개선 사항

1.  **데이터셋 등록 자동화:** 현재는 아티팩트로 저장만 하고 있습니다. MLflow Model Registry를 활용하여 데이터셋을 정식으로 등록하고, 데이터셋 버전을 관리하는 워크플로우를 추가해야 합니다.
2.  **실시간 모니터링:** `DataQualityManager`의 `invalid_ratio` 지표를 Prometheus/Grafana와 연동하여 실시간으로 모니터링합니다. 무효 비율이 특정 임계값을 초과하면 알림을 발생시켜, 로봇 센서 또는 통신 문제에 즉시 대응할 수 있도록 합니다.
3.  **데이터 전처리 파이프라인:** 수집된 JSON 데이터를 학습에 적합한 형태로 변환하는 전처리 파이프라인(예: 시계열 데이터 정규화, 특징 추출)을 MLOps 플랫폼에 통합하여, 데이터 전처리 과정의 자동화 및 버전 관리를 구현해야 합니다.
