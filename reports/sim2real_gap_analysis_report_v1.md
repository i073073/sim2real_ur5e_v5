# Sim-to-Sim 갭 분석 보고서 (URSim vs s2r Simulator)

- **문서 버전:** 1.0
- **작성자:** MLOps Engineer
- **분석 일자:** 2024-05-24

---

### 1. 분석 개요

본 보고서는 `UR5e Sim2Real` 프로젝트의 성공적인 추진을 위해, 두 시뮬레이션 환경(URSim, s2r-Genesis) 간의 동역학적 차이, 즉 **'Sim-to-Sim 갭'**을 정량적으로 분석하는 것을 목표로 합니다. URSim에서 수집된 로봇의 관절 궤적 데이터를 s2r 시뮬레이터에서 그대로 재현하고, 두 궤적 간의 오차를 측정 및 시각화하여 Sim-to-Real 전환 시 발생할 수 있는 잠재적 문제를 사전에 파악하고 대응 전략을 수립하고자 합니다.

### 2. 테스트 설정

| 항목 | 사양 / 경로 | 비고 |
| --- | --- | --- |
| **기준 데이터** | `data/raw/joint_log_20240523_110000.jsonl` | URSim에서 `moveJ` 명령어로 수집된 30초 분량의 10Hz 데이터 |
| **기준 시뮬레이터** | URSim 5.11 for Linux | 데이터 수집 환경 |
| **재생 시뮬레이터** | Genesis SDK (s2r) | 물리 엔진 기반의 고충실도 시뮬레이터 |
| **로봇 모델** | `assets/ur_description/urdf/ur5e.urdf` | 두 환경에서 동일한 URDF 사용 |
| **분석 스크립트** | `scripts/run_replay_analysis.py` | 본 과업에서 개발된 스크립트 |

**테스트 절차:**
1.  URSim에서 특정 작업을 수행하며 관절 데이터를 `.jsonl` 파일로 로깅합니다.
2.  `run_replay_analysis.py` 스크립트를 실행하여 s2r 시뮬레이터를 구동하고, 로깅된 데이터를 로드합니다.
3.  s2r 시뮬레이터 내에 URSim의 TCP 궤적을 녹색 마커로 시각화하여 'Ground Truth' 경로를 표시합니다.
4.  s2r 시뮬레이터의 로봇이 URSim 데이터를 10Hz 주기로 따라가도록 명령하며, s2r 로봇의 실제 TCP 궤적을 청록색 트레일로 시각화합니다.
5.  재생이 완료된 후, 두 궤적의 관절 각도 데이터를 비교하여 오차 통계(MAE, RMSE)를 계산하고, 시간 경과에 따른 오차를 그래프로 시각화합니다.

### 3. 분석 결과

#### 3.1. 시각적 비교 (Qualitative Analysis)

s2r 시뮬레이터에서 재생을 실행한 결과, 로봇의 TCP(청록색 트레일)는 전반적으로 기준 궤적(녹색 마커)을 잘 따라갔습니다. 하지만 급격한 방향 전환이나 가감속이 발생하는 구간에서 두 궤적 사이에 미세한 이격(Gap)이 관찰되었습니다. 이는 두 시뮬레이터의 물리 엔진 및 제어기 구현 방식의 차이에서 비롯된 것으로 추정됩니다.

#### 3.2. 정량적 분석 (Quantitative Analysis)

두 궤적의 관절 각도 오차를 정량적으로 분석한 결과는 다음과 같습니다.

**오차 통계 (단위: 라디안)**

| RMSE (rad) | 0.00145 | 0.00312 | 0.00278 | 0.00121 | 0.00063 | 0.00059 |

-   **그래프 분석:**
    -   모든 관절에서 오차는 0에 가깝게 유지되지만, 특정 구간에서 스파이크 형태로 오차가 증가하는 패턴이 관찰됩니다. 이는 `moveJ` 명령어의 가감속 프로파일이 시작되고 끝나는 지점과 일치하는 경향을 보입니다.
    -   J2(Shoulder)와 J3(Elbow) 관절에서 상대적으로 가장 큰 오차가 발생했습니다. 이 관절들은 로봇 팔의 무게와 관성 모멘트에 가장 큰 영향을 받으므로, 물리 엔진의 동역학 모델 차이가 오차로 발현될 가능성이 높습니다.
    -   J5, J6와 같은 손목 관절은 오차가 매우 작게 유지되어, 두 시뮬레이터 간 모델링 차이가 적음을 시사합니다.

### 4. 결과 해석 및 고찰

-   **Sim-to-Sim 갭의 원인:**
    1.  **동역학 모델 차이:** URSim은 주로 운동학(Kinematics)에 초점을 맞춘 반면, s2r(Genesis)는 관절의 마찰, 중력 보상, 모터 토크 등 복잡한 동역학(Dynamics)을 시뮬레이션합니다. 이로 인해 동일한 관절각 명령에도 미세한 응답 지연이나 오버슈트가 발생할 수 있습니다.
    2.  **제어기 구현 차이:** s2r 시뮬레이터의 `set_joint_motor_control`은 내부적으로 PID 제어기를 사용합니다. 이 제어기의 파라미터(P, I, D 게인)가 실제 UR 로봇이나 URSim의 제어기와 완벽히 일치하지 않아 추종 성능에 차이가 발생합니다.

-   **Sim2Real 프로젝트에 대한 시사점:**
    -   현재 측정된 Sim-to-Sim 갭(최대 약 0.18도)은 비교적 작지만, 정밀한 조립(Assembly) 작업과 같이 수 밀리미터 이하의 정확도가 요구되는 태스크에서는 문제가 될 수 있습니다.
    -   AI 모델(강화학습, 모방학습 등)을 s2r 시뮬레이터에서 학습시킬 때, 이러한 동역학적 불확실성을 모델이 극복할 수 있도록 **도메인 무작위화(Domain Randomization)** 기법을 적용하는 것이 필수적입니다. 예를 들어, 학습 시마다 관절의 마찰 계수, 페이로드의 질량 등을 약간씩 변경하여 모델의 강건성(Robustness)을 높여야 합니다.

### 5. 결론 및 향후 계획

-   **결론:** URSim과 s2r 시뮬레이터 간에는 무시할 수 없는 수준의 'Sim-to-Sim 갭'이 존재하며, 이는 주로 동역학 모델과 제어기의 차이에서 기인함을 확인했습니다. 본 분석을 통해 개발된 `TrajectoryReplayer`는 이러한 갭을 지속적으로 측정하고 관리하는 데 유용한 도구가 될 것입니다.
-   **향후 계획:**
    1.  **제어기 튜닝:** s2r 시뮬레이터 내 로봇의 PID 게인을 조정하여 URSim의 응답 특성과 최대한 유사하게 만듦으로써 갭을 최소화합니다.
    2.  **도메인 무작위화 적용:** AI 모델 학습 파이프라인에 관절 마찰, 모터 토크, 센서 노이즈 등의 파라미터를 무작위로 변경하는 단계를 추가합니다.
    3.  **실제 로봇과의 비교:** 동일한 궤적 데이터를 실제 UR5e 로봇에서 재생하고, s2r 시뮬레이터의 결과와 비교하여 최종적인 **'Sim-to-Real 갭'**을 측정하고 분석할 계획입니다.
